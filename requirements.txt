# =====================================================================
# INSTALLATION & VIRTUAL ENVIRONMENT INSTRUCTIONS (via Terminal)
# =====================================================================
#
# IMPORTANT: Always use the venv Python directly (.venv\Scripts\python.exe)
#            to avoid accidentally using the system Python which lacks packages.
#
# Step 1: Create a Python 3.11/3.12 virtual environment:
#    python -m venv .venv
#
# Step 2: Activate the virtual environment:
#    - Windows (PowerShell):  .\.venv\Scripts\Activate.ps1
#    - Windows (Cmd Prompt):  .\.venv\Scripts\activate.bat
#    - macOS / Linux:         source .venv/bin/activate
#
# Step 3: Upgrade pip to the latest version:
#    .\.venv\Scripts\python.exe -m pip install --upgrade pip
#
# Step 4: Install all required packages:
#    .\.venv\Scripts\pip.exe install -r requirements.txt
#
# Step 5: (Windows Only - Troubleshooting) If PyAudio installation fails:
#    .\.venv\Scripts\pip.exe install pipwin
#    pipwin install pyaudio
#
# =====================================================================
# HOW TO CAPTURE & STORE NEW DEPENDENCIES (pip freeze)
# =====================================================================
#
# WARNING: Do NOT run 'pip freeze > requirements.txt' directly.
#          It will overwrite this file and delete all these instructions!
#
# To save/store updated packages:
#   1. Run:  pip freeze > temp_reqs.txt
#   2. Copy the package lines from 'temp_reqs.txt'
#   3. Paste/replace them in the dependency section at the bottom of this file.
#   4. Delete the temporary file:
#      On Windows:      del temp_reqs.txt
#      On macOS/Linux:  rm temp_reqs.txt
#
# =====================================================================
# VERSION NOTES (torch / pyannote compatibility matrix)
# =====================================================================
#
# pyannote.audio 4.x requires torch >= 2.8.0 AND pytorch-lightning 2.5.x
# pytorch-lightning 2.5.x is incompatible with torch >= 2.12
# Therefore: torch==2.8.0 and torchaudio==2.8.0 are the pinned versions.
# Do NOT upgrade torch beyond 2.11.x unless you also upgrade pytorch-lightning.
#
# =====================================================================
# HOW TO RUN LIVE TRANSCRIPTION & DIARIZATION MODELS (Step-by-Step)
# =====================================================================
#
# Step 1: Create local environment configuration:
#    - Copy the configuration template:
#      On Windows:      copy .env.example .env
#      On macOS/Linux:  cp .env.example .env
#    - Open the new .env file and configure:
#        CLINICAL_COPILOT_ELEVENLABS_API_KEY=<your_key>
#        CLINICAL_COPILOT_STT_PROVIDER_MODE=production
#        CLINICAL_COPILOT_HUGGINGFACE_TOKEN=<your_token> (Required for Pyannote)
#        CLINICAL_COPILOT_DIARIZATION_MODE=production   <- or 'development' for mocks
#
# Step 2: Start the FastAPI Ingestion Server (Terminal 1):
#    .\.venv\Scripts\python.exe -m uvicorn app.main:app --port 8000
#
#    On first run with DIARIZATION_MODE=production, the server will
#    automatically download the Pyannote speaker-diarization-3.1 model
#    (~1 GB from HuggingFace). Subsequent starts use the local cache.
#
# Step 3: Stream Audio to the Server (Terminal 2 — open new terminal, activate venv):
#
#    Option A: Stream Live Microphone Input:
#       .\.venv\Scripts\python.exe stream_mic.py doctor
#
#    Option B: Stream a Pre-recorded WAV File:
#       .\.venv\Scripts\python.exe stream_audio.py CAR0001.wav doctor
#
#    Roles available: doctor | patient | attender
#
# Step 4: Fetch / Verify Captured Transcripts:
#    - The streaming client terminal prints transcripts in real time.
#    - Query the session transcript cache via HTTP at any time:
#      http://127.0.0.1:8000/v1/transcripts/consultation-xyz-123
#
# Step 5: Health Check:
#      http://127.0.0.1:8000/health
#
# =====================================================================
# HOW TO RUN THE CLINICAL EXTRACTION NLP MODEL
# =====================================================================
#
# The clinical extraction NLP model extracts symptoms, medications, diagnoses,
# procedures, and risk factors with negation handling from the dialogue.
#
# Option A: Run interactive CLI & verification tests:
#    .\.venv\Scripts\python.exe verify_step_4.py
#    (This runs batch tests, then prompts you to type sentences in real-time)
#
# Option B: Run complete Unit Test suite:
#    .\.venv\Scripts\python.exe -m unittest discover -s tests -p "test_*.py"
#
# =====================================================================
# DIARIZATION MODES

#                  Requires CLINICAL_COPILOT_HUGGINGFACE_TOKEN in .env.
#                  Downloads pyannote/speaker-diarization-3.1 on first run.
#
